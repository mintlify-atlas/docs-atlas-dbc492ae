ThystTV includes a comprehensive statistics tracking system that monitors user viewing habits, provides analytics, and tracks watch streaks.

## System Overview

The statistics system follows the MVVM pattern with Room database for persistence, coroutines for async operations, and StateFlow for reactive UI updates.

## Architecture Components

### Session Recording

<Accordion title="Session Recording Flow">

1. **Session Start**: User begins watching a stream/video
2. **Tracking**: App tracks channel ID, game, start time, etc.
3. **Session End**: User stops watching or switches content
4. **Recording**: Session data is persisted to `watch_sessions` table
5. **Streak Update**: Repository checks if this is a consecutive day and updates streak

```kotlin
suspend fun recordWatchSession(session: WatchSession) {
    watchSessionDao.insert(session)
    updateWatchStreak(session.date)
}

private suspend fun updateWatchStreak(currentDate: String) {
    val streak = watchStreakDao.getStreak()
    if (streak == null) {
        watchStreakDao.upsert(WatchStreak(1, 1, 1, currentDate, currentDate))
    } else if (streak.lastWatchDate != currentDate) {
        val isConsecutive = isConsecutiveDay(streak.lastWatchDate, currentDate)
        val newCurrent = if (isConsecutive) streak.currentStreakDays + 1 else 1
        val newLongest = maxOf(streak.longestStreakDays, newCurrent)
        watchStreakDao.updateStreak(newCurrent, newLongest, currentDate, ...)
    }
}
```

</Accordion>

### 4. Streamer Loyalty

**Purpose**: Rank streamers by engagement metrics

**Loyalty Score Calculation**:

```kotlin
data class StreamerLoyalty(
    val channelId: String,
    val channelName: String,
    val channelLogin: String,
    val totalWatchSeconds: Long,
    val sessionCount: Int,
    val distinctDaysWatched: Int,
    val loyaltyScore: Double
) {
    companion object {
        fun calculateScore(
            watchSeconds: Long,
            sessions: Int,
            days: Int,
            maxWatchSeconds: Long,
            maxSessionCount: Int,
            maxDistinctDays: Int
        ): Double {
            // Weighted average of normalized metrics
            val watchWeight = 0.4
            val sessionWeight = 0.3
            val daysWeight = 0.3
            
            return (watchSeconds.toDouble() / maxWatchSeconds * watchWeight) +
                   (sessions.toDouble() / maxSessionCount * sessionWeight) +
                   (days.toDouble() / maxDistinctDays * daysWeight)
        }
    }
}
```

<Info>
Loyalty score considers watch time (40%), session count (30%), and distinct days watched (30%).
</Info>

### 5. Watch Streak

**Purpose**: Gamification - encourage daily viewing

**Streak Logic**:
- **Current Streak**: Consecutive days with at least one watch session
- **Longest Streak**: All-time record
- **Streak Break**: Missing a day resets current streak to 1

```kotlin
private fun isConsecutiveDay(lastDate: String, currentDate: String): Boolean {
    val sdf = SimpleDateFormat("yyyy-MM-dd", Locale.US)
    val last = sdf.parse(lastDate) ?: return false
    val current = sdf.parse(currentDate) ?: return false
    val diffMs = current.time - last.time
    val diffDays = diffMs / (1000 * 60 * 60 * 24)
    return diffDays == 1L
}
```

## Performance Optimizations

### 1. Database Indexing

Strategic indexes on frequently queried columns:

```sql
CREATE INDEX index_watch_sessions_date ON watch_sessions (date)
CREATE INDEX index_watch_sessions_channelId ON watch_sessions (channelId)
CREATE INDEX index_watch_sessions_gameId ON watch_sessions (gameId)
```

### 2. Flow Conversion

Convert cold Flow to hot StateFlow with replay:

```kotlin
val screenTime: StateFlow<List<ScreenTime>> = repository.getAllScreenTime()
    .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), emptyList())
```

**Benefits**:
- 5-second timeout prevents unnecessary database queries
- Multiple observers share the same Flow
- Automatic lifecycle handling

### 3. Batch Operations

Update multiple statistics atomically:

```kotlin
suspend fun recordWatchSession(session: WatchSession) {
    watchSessionDao.insert(session)
    updateWatchStreak(session.date)
    // Both operations in same transaction context
}
```
