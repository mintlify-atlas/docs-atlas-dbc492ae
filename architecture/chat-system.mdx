ThystTV's chat system supports both live chat via WebSocket/IRC and chat replay for VODs, with extensive third-party emote integration and real-time event handling.

## System Overview

The chat system is a complex, event-driven architecture that handles:
- Live chat via WebSocket or IRC
- Chat replay synchronized with video playback
- Third-party emotes (7TV, BTTV, FFZ)
- PubSub events (raids, polls, predictions)
- Message translation and moderation features

**Location**: `util/chat/ChatWriteWebSocket.kt`

**Location**: `util/chat/ChatReadIRC.kt`, `util/chat/ChatWriteIRC.kt`

**Synchronization Logic**:

```kotlin
while (isActive) {
    val message = list.firstOrNull() ?: break
    val messageOffset = message.offsetSeconds.times(1000)
    val currentPosition = getCurrentPosition() ?: 0 + startTime
    
    while (currentPosition < messageOffset) {
        // Calculate delay based on playback speed
        delay(max((messageOffset - currentPosition) / playbackSpeed, 0))
    }
    
    // Emit message at correct time
    listener.onChatMessage(message.toChatMessage())
    list.remove(message)
}
```

## Emote System

### Emote Sources

1. **Twitch Emotes**
   - Global emotes (available to all users)
   - Subscriber emotes (channel-specific)
   - User emotes (followed channels, Turbo, Prime)

2. **7TV (SevenTV)**
   - Global emote set
   - Channel-specific emotes
   - Personal emote sets
   - Animated emotes (WebP support)

3. **BTTV (BetterTTV)**
   - Global emotes
   - Channel emotes

4. **FFZ (FrankerFaceZ)**
   - Global emotes
   - Channel emotes

### Emote Loading Strategy

**Optimization**: Cache global emotes to reduce API calls

```kotlin
private var savedGlobalStvEmotes: List<Emote>? = null
private var savedGlobalBttvEmotes: List<Emote>? = null
private var savedGlobalFfzEmotes: List<Emote>? = null

val saved = savedGlobalStvEmotes
if (!saved.isNullOrEmpty()) {
    synchronized(thirdPartyEmotes) {
        thirdPartyEmotes.addAll(saved)
    }
} else {
    viewModelScope.launch {
        val emotes = playerRepository.loadGlobalStvEmotes()
        savedGlobalStvEmotes = emotes
        synchronized(thirdPartyEmotes) {
            thirdPartyEmotes.addAll(emotes)
        }
    }
}
```

### Emote Resolution

Chat messages are parsed to replace emote codes with renderable emotes:

1. **Twitch Emotes**: Included in IRC tags with positions
2. **Third-Party Emotes**: Text matching against loaded emote names
3. **Zero-Width Emotes**: Overlay emotes that render on top of others

## Message Buffer Management

**Buffer Limit**: Configurable (default 600 messages)

**Overflow Handling**:

```kotlin
suspend fun onMessage(message: ChatMessage) {
    synchronized(chatMessages) {
        chatMessages.add(message)
        val removeCount = if (chatMessages.size > messageLimit) {
            chatMessages.size - messageLimit
        } else 0
        
        if (removeCount > 0) {
            repeat(removeCount) {
                chatMessages.removeAt(0)  // Remove oldest
            }
        }
    }
}
```

<Info>
Old messages are removed from the top of the list to maintain buffer size. RecyclerView is notified to remove items efficiently.
</Info>

## Connection Lifecycle

### Start Live Chat

```kotlin
fun startLive(channelId: String?, channelLogin: String?, streamId: String?) {
    messageLimit = prefs.getInt(C.CHAT_LIMIT, 600)
    this.streamId = streamId
    
    // Establish connections
    startLiveChat(channelId, channelLogin)
    
    // Load emotes
    loadEmotes(channelId, channelLogin)
    
    // Load recent messages
    if (prefs.getBoolean(C.CHAT_RECENT, true)) {
        loadRecentMessages(channelLogin)
    }
    
    // Load user emotes if logged in
    if (isLoggedIn) {
        loadUserEmotes(channelId)
    }
}
```

### Disconnect

```kotlin
fun disconnect() {
    stopLiveChat()
    chatMessages.clear()
    autoReconnect = false
    onMessage(ChatMessage(systemMsg = "Disconnected"))
}
```

### Auto-Reconnect

**Trigger**: App resumes from background

```kotlin
fun resumeLive(channelId: String?, channelLogin: String?) {
    if (chatReadJob?.isActive == false && autoReconnect) {
        startLiveChat(channelId, channelLogin)
    }
}
```

## Performance Optimizations

### 1. Concurrent Hash Map for Chatters

```kotlin
private val chatters = ConcurrentHashMap<String, Chatter>()
```

**Benefits**: Thread-safe access without explicit synchronization overhead

### 2. Synchronized Collections

Critical shared state is protected:

```kotlin
synchronized(chatMessages) {
    chatMessages.add(message)
    // ...
}
```

### 3. Lazy Emote Loading

```kotlin
val recentEmotes by lazy { playerRepository.loadRecentEmotesFlow() }
```

### 4. RecyclerView Optimization

- `itemAnimator = null` - Disable default animations for performance
- `stackFromEnd = true` - Start from bottom (chat convention)
- Efficient notify methods (`notifyItemInserted`, `notifyItemRangeRemoved`)
