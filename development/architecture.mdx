## Project Structure

The app follows a clean architecture pattern with clear separation of concerns:

```
com.github.andreyasadchy.xtra/
├── db/                    # Room database and DAOs
├── di/                    # Hilt dependency injection modules
├── model/                 # Data models and entities
│   ├── chat/             # Chat-related models
│   ├── gql/              # GraphQL models
│   ├── helix/            # Twitch Helix API models
│   ├── stats/            # Screen time and statistics models
│   └── ui/               # UI-specific models
├── player/               # Media player components
├── repository/           # Data repositories
│   └── datasource/       # Remote and local data sources
├── ui/                   # UI layer (Fragments, Activities, ViewModels)
│   ├── channel/          # Channel browsing
│   ├── chat/             # Chat and floating chat overlay
│   ├── player/           # Video player and gesture controls
│   ├── stats/            # Screen time statistics
│   ├── settings/         # App settings
│   └── ...
└── util/                 # Utility classes and helpers
```

## Core Architecture Layers

### 1. Data Layer

#### Room Database

The app uses Room for local persistence with 19+ specialized DAOs:

<CodeGroup>
```kotlin Key DAOs
// Screen time tracking
ScreenTimeDao
WatchSessionDao
WatchStreakDao
StreamWatchStatsDao

// User data
BookmarksDao
VideoPositionsDao
VideosDao

// Following and favorites
LocalFollowsChannelDao
LocalFollowsGameDao

// Search and filters
RecentSearchDao
SavedFiltersDao
```
</CodeGroup>

<Note>
  Room schema location is configured at `app/build.gradle.kts:157`. Database migrations are handled in `db/AppDatabase.kt`
</Note>

#### Repositories

Repositories abstract data sources and provide a clean API for ViewModels. They handle:
- Remote API calls (Twitch Helix API, GraphQL)
- Local database operations
- Data caching and synchronization
- Paging support for lists

### 2. Dependency Injection

Hilt is used throughout the app for dependency injection:

<CodeGroup>
```kotlin Module Files
// di/DatabaseModule.kt - Provides Room database and DAOs
@Module
@InstallIn(SingletonComponent::class)
object DatabaseModule

// di/ThystTVModule.kt - Provides network clients, repositories
@Module
@InstallIn(SingletonComponent::class)
object ThystTVModule
```
</CodeGroup>

### 3. UI Layer

The UI follows the MVVM (Model-View-ViewModel) pattern:

- **Fragments**: Display UI and handle user interactions
- **ViewModels**: Manage UI state and business logic
- **ViewBinding**: Type-safe view references
- **Navigation Component**: Fragment navigation with Safe Args

<Note>
  ViewBinding is enabled at `app/build.gradle.kts:65`. Navigation Safe Args plugin is applied at `app/build.gradle.kts:8`
</Note>

## Key Features Architecture

### Gesture System

The gesture control system is a sophisticated state machine for video player interactions:

<Steps>
  <Step title="PlayerGestureListener">
    Central state machine extending `GestureDetector.SimpleOnGestureListener`
    
    - Detects gesture types based on screen zones
    - Manages gesture lifecycle (Down → Scroll → Up/Cancel)
    - Prevents conflicts with other touch handlers
  </Step>

  <Step title="PlayerGestureHelper">
    Stateless helper class for calculations:
    
    - Volume/brightness value calculations
    - Time duration formatting
    - Zone boundary detection
    - Swipe direction determination
  </Step>

  <Step title="PlayerGestureCallback">
    Interface for player control:
    
    - Seek, playback speed control
    - Controller visibility
    - Window attributes (brightness)
    - Visual feedback overlay
  </Step>
</Steps>

<Note>
  Detailed gesture system documentation is available at `source/docs/GESTURE_SYSTEM.md`
</Note>

### Screen Time Tracking

Screen time statistics use a multi-DAO approach:

```kotlin
WatchSessionDao       // Individual viewing sessions
ScreenTimeDao         // Aggregated daily screen time
StreamWatchStatsDao   // Per-channel statistics
WatchStreakDao        // Viewing streaks
```

Data is:
- Stored locally in Room database
- Never shared externally (privacy-first)
- Visualized with charts in the Stats tab

### Floating Chat

The floating chat overlay uses:
- Custom ViewGroup for draggable, resizable overlay
- WindowManager for system-level overlay
- SharedPreferences for position/size persistence
- High contrast mode for readability

## Network Layer

ThystTV communicates with Twitch using multiple protocols:

<Tabs>
  <Tab title="REST API">
    **Twitch Helix API** via OkHttp + Cronet
    
    - Channel data, streams, videos
    - User information
    - Game categories
    
    Dependencies: `libs.okhttp`, `libs.cronet.api`
  </Tab>
  
  <Tab title="GraphQL">
    **Twitch GraphQL API** via Apollo
    
    - Advanced queries
    - Subscriptions
    - Real-time data
    
    Configuration: `app/build.gradle.kts:160-164`
  </Tab>
  
  <Tab title="Chat (IRC)">
    **Twitch Chat IRC** with custom implementation
    
    - Real-time chat messages
    - Emotes and badges
    - Chat commands
  </Tab>
</Tabs>

## Media Playback

Video playback is powered by **Media3 ExoPlayer**:

```kotlin
implementation(libs.media3.exoplayer)
implementation(libs.media3.exoplayer.hls)  // HLS streaming
implementation(libs.media3.ui)             // Player UI
implementation(libs.media3.session)        // Background playback
```

Features:
- HLS adaptive streaming
- Picture-in-Picture (PiP)
- Background audio playback
- Playback speed control
- Seek preview thumbnails

## Security & Privacy

### EncryptedSharedPreferences

Sensitive data is encrypted using AndroidX Security:

```kotlin
implementation(libs.security.crypto)
```

### TLS/SSL

Network security with Conscrypt:

```kotlin
implementation(libs.conscrypt)
```

### Local-Only Data

All user statistics and preferences are:
- Stored locally in Room database
- Never transmitted to external servers
- Fully under user control

## Performance Optimizations

<AccordionGroup>
  <Accordion title="ProGuard/R8">

    - Code shrinking
    - Obfuscation
    - Optimization
    
    Configured at `app/build.gradle.kts:52-54`

  <Accordion title="Resource Shrinking">

    ```kotlin
    isShrinkResources = true
    ```
  </Accordion>

  <Accordion title="JNI Library Exclusion">
    Unused translation JNI libraries are excluded to reduce APK size:
    ```kotlin
    packaging.jniLibs.excludes.addAll(listOf(
        "lib/x86/libtranslate_jni.so",
        "lib/x86_64/libtranslate_jni.so",
        // ...
    ))
    ```
  </Accordion>

  <Accordion title="Paging 3">
    Large lists use Paging 3 for efficient loading:
    - Streams, videos, channels
    - Search results
    - Chat history
  </Accordion>
</AccordionGroup>
