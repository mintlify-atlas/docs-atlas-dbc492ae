ThystTV provides extensive customization options to tailor your viewing experience. Access settings by tapping the menu icon and selecting "Settings".

## Player Settings

### Gesture Controls

<Info>
Gesture controls allow you to adjust volume, brightness, seek, and playback speed without blocking the video.
</Info>

<Accordion title="Gesture Configuration">

**Enable/Disable Gestures**
- Toggle gesture controls on or off
- Gestures are automatically disabled when player controls are visible

**Haptic Feedback**
- Optional vibration feedback during gesture adjustments
- Disabled by default to preserve battery

**Sensitivity**
- Adjust how responsive gestures are to your swipes
- Options: Low, Normal (default), High

**Zone Split**
- Configure the boundary between top/bottom gesture zones
- Affects seek vs. playback speed gesture areas

</Accordion>

### Video Playback

<Accordion title="Quality & Performance">

**Default Quality**
- Set preferred stream quality (Auto, 1080p60, 720p60, 480p, 360p, 160p, Audio Only)
- Default: Saved (remembers last selection)

**Data Saver**
- Automatically lower quality on cellular connections
- Configure separate cellular quality preference

**Background Playback**
- Continue audio when app is minimized
- Picture-in-Picture mode support
- Background audio when screen is locked

**Video Position Saving**
- Automatically resume VODs where you left off
- Can be cleared via "Delete video positions"

</Accordion>

<Accordion title="Advanced Player Options">

**Audio Settings**
- Audio focus (pause when other apps play audio)
- Handle audio becoming noisy (pause when headphones disconnect)
- Background audio track support

**Display Options**
- Disable background video rendering
- Keep screen on when paused
- Avoid rounded corners (padding for notched displays)
- Move player freely in landscape mode

</Accordion>

### Buffer Settings

<Warning>
Only modify buffer settings if you experience playback issues. Incorrect values can cause stuttering or increased latency.
</Warning>

Configure advanced buffering parameters:

- **Min Buffer**: 15000ms (default)
- **Max Buffer**: 50000ms (default)
- **Buffer for Playback**: 2000ms (default)
- **Rebuffer**: 2000ms (default)
- **Live Target Offset**: 2000ms (default)
- **Live Min/Max Speed**: Control playback speed adjustments for live streams

### Player Menu Customization

Toggle which options appear in the player menu:

- Quality selector
- Playback speed
- Viewer list
- Chapters
- Download
- Bookmark
- Sleep timer
- Aspect ratio
- Volume
- Subtitles toggle
- Restart player
- Chat bar toggle
- Chat toggle
- Reload emotes
- Disconnect chat

## Chat Settings

### Appearance

<Accordion title="Message Display">

**Username Styling**
- Bold usernames
- Random colors for usernames without set colors
- Theme-adapted username colors for better contrast

**Timestamps**
- Show/hide message timestamps
- Format options: 12-hour, 24-hour

**Size Adjustments**
- Text size (default: 14)
- Emote size (default: 29.5)
- Badge size (default: 18.5)
- Chat size modifier: 1-200% (default: 100%)
- Landscape chat width: 0-100% (default: 30%)

</Accordion>

### Emotes & Images

<Accordion title="Emote Providers">

**Third-Party Emote Support**
- 7TV (SevenTV)
- BTTV (BetterTTV)
- FFZ (FrankerFaceZ)

**7TV Features**
- Personal emotes
- Live emote updates
- Name paints (custom username colors)
- 7TV badges

**Emote Settings**
- Animated GIF emotes
- Zero-width (overlay) emotes
- System message emotes

</Accordion>

<Accordion title="Image Quality">

**Image Library**
- Choose rendering library for emotes
- Options affect performance and compatibility

**WebP Support**
- Use WebP format for better compression
- Reduces data usage

**Image Quality**
- Scale: 1x, 2x, 3x, 4x (default: 4x)
- Higher quality uses more bandwidth

</Accordion>

### Chat Behavior

<Accordion title="Message Limits">

**Message Limit**
- Maximum messages stored: 1-1000 (default: 600)
- Higher limits use more memory

**Recent Messages**
- Show recent messages when joining chat
- Limit: 1-800 (default: 100)

</Accordion>

<Accordion title="Interactive Features">

**PubSub Events**
- Channel point redemptions
- Automatically collect channel points
- Point collection notifications

**Stream Events**
- Show raid notifications
- Auto-switch to raided channel
- Show polls
- Show predictions

**Chat Moderation**
- Show deleted messages
- Show cleared chat notifications
- Show user notices (subs, resubs, etc.)
- First message visibility options

</Accordion>

### Translation

<Info>
ThystTV includes offline translation powered by on-device ML models.
</Info>

- Enable/disable chat message translation
- Set target language (default: English)
- Manage downloaded language models
- Translate messages on-demand or automatically

### Connection Settings

<Accordion title="Advanced Chat Options">

**Protocol**
- Use WebSocket (recommended)
- Use SSL/TLS encryption
- Enable PubSub for advanced features

**Chat Overlay**
- Keep chat open during fullscreen
- Floating chat transparency: 0-100%
- High visibility mode for better readability

**Other**
- Disable chat entirely

</Accordion>

## Network Settings

### Proxy Configuration

<Warning>
Proxy settings are for advanced users. Incorrect configuration may prevent streams from loading.
</Warning>

<Accordion title="Stream Proxy">

**Custom Proxy**
- Enable custom stream proxy
- Default: `https://api.ttv.lol/playlist/$channel.m3u8`
- Supports playlist manipulation and alternative backends

**Additional Proxy Settings**
- Configure advanced proxy parameters
- Custom headers for stream requests

</Accordion>

### Playback Token Settings

<Accordion title="Advanced Playback Configuration">

**Stream Tokens**
- Supported codecs (default: av1,h265,h264)
- Device ID (randomized or custom)
- Player type (default: site)
- Include GQL token for subscriber-only streams
- Custom stream headers

**VOD Tokens**
- Video player type (default: channel_home_live)
- Include GQL token for subscriber content
- Skip video access token options

**Ad Blocking**
- Hide player during ads (experimental)

</Accordion>
